# 文件导入与浅色主题切换规格

## Why
当前工具只能手动粘贴代码，缺少“选择/拖入本地文件”的便捷入口，且界面固定为暗色风格，缺少浅色主题切换按钮，影响不同场景下的可用性与可读性。

## What Changes
- 在页面顶部操作区新增“选择本地文件”按钮和隐藏的文件选择器，支持单文件导入。
- 支持将本地代码文件拖拽到页面任意区域自动导入，提供可视拖拽提示。
- 导入时根据文件后缀自动匹配已有语言映射，并回退到 `hljs.highlightAuto` 推断语言后联动语言选择器。
- 导入成功后将文件内容写入编辑器并刷新预览、统计与滚动位置。
- 新增浅色主题模式开关按钮（仅影响应用壳层 UI），并在切换时保持预览区代码高亮主题不受影响。
- 通过 `localStorage` 记住上次选择的语言与浅/暗模式偏好，页面重启后恢复。

## Impact
- Affected specs: 无（新增能力）
- Affected code: `code-highlighter.html`

## ADDED Requirements
### Requirement: 文件选择与导入
系统 SHALL 提供显式按钮和隐藏 `<input type="file">`，支持选择本地代码文件并导入。

#### Scenario: 通过按钮选择文件
- **WHEN** 用户点击“选择文件”按钮
- **THEN** 系统打开文件选择框，用户选择文件后将内容读取并填入编辑器

#### Scenario: 文件读取失败
- **WHEN** 用户选择的文件无法读取（例如空文件或读取异常）
- **THEN** 系统显示错误提示，不改变编辑器内容

### Requirement: 拖拽导入文件
系统 SHALL 支持将文件拖拽到页面任意区域进行导入，并提供可视反馈。

#### Scenario: 拖入代码文件
- **WHEN** 用户将一个文件拖入页面并释放
- **THEN** 系统读取文件内容，自动检测语言并更新编辑器与预览

#### Scenario: 拖拽悬停反馈
- **WHEN** 用户将文件拖入页面区域
- **THEN** 页面出现可视拖拽提示层

### Requirement: 自动语言检测与同步
系统 SHALL 根据文件后缀映射到已有语言列表；当后缀无法命中时，使用 `hljs.highlightAuto` 推断语言。

#### Scenario: 后缀命中已有语言
- **WHEN** 文件后缀在语言映射表中命中
- **THEN** 系统将语言选择器切换到命中语言

#### Scenario: 后缀未命中
- **WHEN** 文件后缀未命中已知语言
- **THEN** 系统执行自动检测，若命中则切换语言；否则保持当前语言并提示未知语言

### Requirement: 浅色主题模式开关
系统 SHALL 提供一个浅色主题切换按钮，用于切换应用壳层 UI 的明暗模式。

#### Scenario: 切换到浅色模式
- **WHEN** 用户点击主题模式开关
- **THEN** 页面背景、顶部栏、底部栏与编辑器面板切换为浅色样式

#### Scenario: 保持预览主题独立
- **WHEN** 用户切换浅/暗模式
- **THEN** 代码高亮主题保持不变，预览区仅跟随主题配色，不随壳层明暗自动改变

### Requirement: 用户偏好记忆
系统 SHALL 使用 `localStorage` 记住语言选择与浅/暗模式偏好。

#### Scenario: 刷新后恢复
- **WHEN** 用户刷新页面
- **THEN** 系统恢复上次选择的语言与浅/暗模式
